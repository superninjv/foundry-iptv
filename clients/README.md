# Foundry IPTV — Native Clients

Three subprojects that form the native client layer for Foundry IPTV.
All authenticate against the household's Next.js server using a **device bearer token**.

## Subprojects

| Directory | Language | Target | Playback |
|-----------|----------|--------|----------|
| `foundry-core/` | Rust | Library (all targets) | N/A — shared logic only |
| `foundry-tv/` | Rust | Linux x86_64, aarch64 | GStreamer playbin3 |
| `foundry-android/` | Kotlin + Rust JNI | Android / Fire TV | ExoPlayer (Media3) |

## Device Token Auth Flow

1. Admin opens the web app and navigates to **Settings → Devices → Add device**.
   A short-lived numeric pairing code is shown (generated by `POST /api/auth/device-token`
   on the server — see Track D).
2. On first launch, the native client prompts for the **server URL** and **pairing code**.
3. The client `POST`s `{ "code": "...", "label_hint": "Living Room TV" }` to
   `/api/auth/device-token` and receives a long-lived raw token string.
4. The token is stored in the platform config file:
   - Linux: `~/.config/foundry-tv/config.toml`
   - Android: `SharedPreferences` (encrypted)
5. Every subsequent API request includes `Authorization: Bearer <token>`.

## Building

### foundry-core (no system deps)

```bash
cd clients
cargo check -p foundry-core
cargo test -p foundry-core
```

### foundry-tv (Linux, requires GStreamer + Slint headers)

```bash
sudo apt install libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev \
    libfontconfig-dev libxkbcommon-dev
cd clients
cargo build -p foundry-tv
```

See `foundry-tv/README.md` for cross-compile instructions (aarch64 Pi 5).

### foundry-android (requires Android Studio + NDK)

See `foundry-android/README.md`. The Kotlin shell cannot be built until the
`foundry-core` JNI libraries are cross-compiled and placed in `foundry-android/jniLibs/`.

## Architecture

```
foundry-core (Rust library)
  ├── api.rs       — typed HTTP client against the Next.js API
  ├── auth.rs      — device token pairing + Bearer injection
  ├── epg.rs       — SQLite-backed EPG cache (SQLx)
  ├── state.rs     — redux-style screen/state machine
  ├── keymap.rs    — platform-agnostic key Intent enum
  └── stream.rs    — StreamSession + StreamQuality types

foundry-tv (Rust binary)
  ├── build.rs     — compiles Slint .slint files
  ├── ui/app.slint — Slint UI definition
  └── src/main.rs  — event loop, tracing init

foundry-android (Kotlin shell)
  ├── MainActivity.kt  — Compose TV entry point
  ├── FoundryCore.kt   — stub for uniffi JNI bindings
  └── jniLibs/         — cross-compiled .so files (not in git)
```
